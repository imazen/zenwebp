## What each is

- **paired_sweep_v2_deblock** тАФ authoritative calibration sweep. 10
  lossless refs ├Ч 16 synthetic source qualities ├Ч 10 targets ├Ч 4
  strategies = 6,400 cells. Columns: `input_path, input_bytes, width,
  height, source_q, source_quantizer_index, encoder_family, source_kind,
  has_alpha, target_zensim_a, strategy, output_bytes, size_ratio,
  measured_zensim_a_vs_source, measured_zensim_a_vs_reference,
  result_kind, error`. Backs `REENCODE` + `REENCODE_GEN_LOSS` in
  `data.rs`. Methodology: `paired_sweep_v2_deblock_2026-05-28.meta.md`.

- **deblock_weakfilter** тАФ `--source-filter 0` rerun proving the deblock
  strategy is net-negative even on un-loop-filtered sources. Methodology +
  conclusion: `deblock_experiment_2026-05-28.md`.

- **test_corpus** тАФ generation-loss-only sweep (`--sources` mode) on the
  existing zenwebp test WebPs. 630 cells, no cumulative column. Early
  smoke validation.

## Regenerate

```bash
cargo build -p zwr-calibrate --release
# Authoritative calibration sweep:
target/release/zwr-calibrate --refs <dir of lossless WebP/PNG refs> \
  --q-grid 20:95:5 --targets 50:95:5 --strategies remux,reencode,deblock,vp8l \
  --output benchmarks/paired_sweep_<date>.csv
# Deblock falsification (weak-filter sources):
target/release/zwr-calibrate --refs <refs> --q-grid 20:95:5 \
  --targets 50:95:5 --strategies reencode,deblock --source-filter 0 \
  --output benchmarks/deblock_weakfilter_<date>.csv
```
